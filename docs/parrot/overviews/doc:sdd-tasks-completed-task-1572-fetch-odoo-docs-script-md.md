## Context

Implements **Module 2** of the spec. Generates the official Odoo 16 / 18 / 19
documentation as PDFs from the official documentation repository, ready for the
PageIndex builder (TASK-1573). Resolves G2 / AC4 (sourcing half).

---

## Scope

- Add `scripts/odoo_agent/fetch_odoo_docs.sh` that, for each version in
  `16.0 18.0 19.0`:
  1. Clones (or reuses a cached clone of) `https://github.com/odoo/documentation.git`.
  2. `git checkout <version>`.
  3. Runs `make latexpdf`.
  4. Copies the produced PDF into `agents/odoo_agent/documentation/<version>/`.
- Be idempotent / re-runnable (skip a version whose PDF already exists unless
  `--force`).
- Fail clearly if the LaTeX toolchain (`make latexpdf` → needs a TeX distribution)
  or network is missing — print the prerequisite and exit non-zero.
- Document the toolchain prerequisites and usage in a header comment and/or
  `scripts/odoo_agent/README.md`.

**NOT in scope**: ingesting the PDFs into a PageIndex (TASK-1573); the agent
(TASK-1574). Do not commit the large generated PDFs unless explicitly decided —
see spec §7 (Git LFS / tracked path consideration).

## Implementation Notes

### Key Constraints
- POSIX-ish bash; `set -euo pipefail`.
- Locate the generated PDF robustly (the latex build output path may vary by
  Sphinx version) — search the build dir for `*.pdf` after `make latexpdf`.
- Parameterize versions in an array so adding a version is one edit.
- Keep clones in a cache dir (e.g. `scripts/odoo_agent/.cache/documentation`) and
  reuse across versions via `git checkout` to avoid re-cloning per version.

### References in Codebase
- `scripts/` — existing helper-script conventions in this repo.
