# F004: Leaf-node body access

Two access paths exist:

1. **NodeContentStore sidecars**: `<storage_dir>/<tree_name>/<node_id>.md` -- loaded via
   `content_loader` closure from `NodeContentStore.loader_for(tree_name)`.
2. **Inline `text` field** on the node dict: fallback when no content_loader
   (`node.get("text") or ""`).

Summary accessed via `node.get("summary") or node.get("prefix_summary") or ""`.

The retriever (retriever.py:104) also checks `node.get("text")` first, then falls back
to summary. The `pageindex://` URI scheme exists in loader.py but is for document
loading, not search.
